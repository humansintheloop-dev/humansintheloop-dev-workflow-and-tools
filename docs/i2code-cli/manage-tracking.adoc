= `i2code manage-tracking` -- HITL Tracking Management
:toc:
:toclevels: 2

== Overview

The `manage-tracking` command manages HITL tracking directories used for session and issue recording.

== Usage

[source,shell]
----
i2code manage-tracking [OPTIONS]
----

At least one of `--migrate` or `--link` must be specified.

== Options

`--migrate`::
Move `.claude/issues` and `.claude/sessions` to `.hitl/` and update `.gitignore`.

`--link DIR`::
Symlink `.hitl/issues` and `.hitl/sessions` to `DIR/issues` and `DIR/sessions`.
This allows sharing tracking data across projects or storing it in a central location.

`--dry-run`::
Show what would be done without making changes.

== Examples

[source,shell]
----
# Migrate from .claude/ to .hitl/
i2code manage-tracking --migrate

# Preview migration without making changes
i2code manage-tracking --migrate --dry-run

# Migrate and link to a shared tracking directory
i2code manage-tracking --migrate --link ~/hitl-tracking

# Link to a shared directory (after migration)
i2code manage-tracking --link ~/hitl-tracking
----
