`pid.yaml` (in a project's root directory) contains important information about a project including what variables to pass to an isolated environment.

Here is an example

```yaml
isolarium:
  container:
    isolation_scripts:
      - path: scripts/container/install-go.sh
      - path: scripts/container/install-linters.sh
      - path: scripts/container/install-pre-commit.sh
      - path: scripts/container/install-codescene.sh
        env:
          - CS_ACCESS_TOKEN
          - CS_ACE_ACCESS_TOKEN
  vm:
    isolation_scripts:
      - path: scripts/vm/install-go.sh
      - path: scripts/vm/install-linters.sh
      - path: scripts/vm/install-pre-commit.sh
      - path: scripts/vm/install-codescene.sh
        env:
          - CS_ACCESS_TOKEN
          - CS_ACE_ACCESS_TOKEN
```

I want it to have an optional digital signature to ensure that the file has not been tampered with. 
This should be in a second YAML document in the same file:

```yaml
...
---
signature:
  algorithm: RSA-SHA256
  key_id: my-key-id
  signature: <base64-encoded-signature>
```
The signature would be generated by taking the first document (the `isolarium` configuration), serializing it to a canonical form, and then signing it with a private key. The `key_id` would be used to identify which public key should be used to verify the signature.

When the file is read, the system would first verify the signature using the specified algorithm and public key. If the signature is valid, it can trust that the configuration has not been altered since it was signed. If the signature is invalid, it should reject the file as potentially tampered with.

The signing key will be in an environment variable/.env.local file - PID_FILE_SIGNING_KEY. This way, the signing key is not stored in the file itself, and can be rotated or changed without modifying the file.

Add a subcommand to `i2code`:

* `i2code pid sign`
* `i2code pid verify-signature`

This would allow users to easily sign their `pid.yaml` files and verify their signatures using the command line tool. The `sign` command would take the path to the `pid.yaml` file, read the first document, generate the signature, and append the second document with the signature information. The `verify-signature` command would read the file, extract the first document and the signature, and verify it using the specified algorithm and public key.
